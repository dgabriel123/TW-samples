Introduction to JSON

JSON, short for JavaScript Object Notation, is a commonly used format for storing and moving information. It is both human-readable and easily parsed by computers, making it a natural fit for the world of REST APIs.

If you already have a working knowledge of JavaScript, JSON will be familiar territory, and you may skip this section. If you are new to JavaScript, read on.

Consider the following example:

        
  {
   "songs": {
    "Kashmir": "Led Zeppelin",
    "Taxman": "The Beatles",
    "Satisfaction": "The Rolling Stones"
   }
  }

All JSON objects are nested inside a set of curly brackets. From this example, we can surmise the following:

  • This example contains one object, titled songs
  • songs contains 3 key-value pairs, which include the song name and the original recording artist

Key-value pairs are the drivers of JSON’s data architecture. In fact, songs is the first key-value pair in this example, and contains 3 pairs inside it.

In JSON, the value is allowed to be 6 types of values: a string, a number, a boolean, an array, a nested object, or null.

String "This is a string", "This is also a string"

A string is a combination of characters. Because JSON strings are wrapped in double quotes, double quotes inside the string need to be escaped as such: "\"".
For example:

"He made \"air quotes"\ with his fingers."
Number 2, 68, 456

A number does not use quotes. If you wrap a numeric value in double quotes, it becomes a string.
Boolean True, False

A boolean is the on/off switch of programming: it can only be True or False.
Array ["String string", 44, False]

An array is a list of items and is marked by straight brackets. These items can be a combination of any of the aforementioned elements, and are separated by a comma.
Nested Object {"Nested object": "You can nest as many objects as you like"}

As I previously mentioned, JSON is a series of nested objects, the root object being JSON itself. You may nest as many objects as you like.
Null Null

A JSON object can contain any and every single one of these elements. The key must be a string, but the value can be any of the 5 elements. Let’s look at a more complex example of a JSON object:

        
{
   "guitar": {
      "style": ["acoustic", "electric"],
      "strings": [6, 7, 8],
      "bridge": ["fixed", "tremolo"],
      "body": ["hollow", "semi-hollow", "solid"],
      "cymbal": False,
      "tunings": {
         "6 string": "EBGDAE",
         "7 string": "EBGDAEB",
         "8 string": "EBGDAEBF#"
      }
   }
}

This JSON example shows the data for the object guitar and contains 6 key-value pairs:

  • style, strings, bridge, and body are arrays
  • cymbal is a boolean value and is set to False
  • tunings contains a nested object with 3 key-value pairs of its own: 6 string, 7 string, and 8 string

Introduction to YAML

.

YAML stands for YAML Ain’t Markup Language. It is even easier to read than JSON, and is the preferred language when working within the OpenAPI spec.

JSON and YAML are used interchangeably, and can be easily converted back and forth with readily available online tools, such as JSON to YAML Converter byBeautify. In this section, I will provide concurrent samples in both languages.

Let’s revisit the JSON in the previous example:

        
{
   "guitar": {
      "style": ["acoustic", "electric"],
      "strings": [6, 7, 8],
      "bridge": ["fixed", "tremolo"],
      "body": ["hollow", "semi-hollow", "solid"],
      "cymbal": False,
      "tunings": {
         "6 string": "EBGDAE",
         "7 string": "EBGDAEB",
         "8 string": "EBGDAEBF#"
      }
   }
}

Here is what it would look like when converted to YAML:

        
guitar:
  style:
    - acoustic
    - electric
  strings:
    - 6
    - 7
    - 8
  bridge:
    - fixed
    - tremolo
  body:
    - hollow
    - semi-hollow
    - solid
  cymbal: False
  tunings:
    6 string: EBGDAE
    7 string: EBGDAEB
    8 string: EBGDAEF#

The syntax is not too dissimilar to JSON, and requires only a brief analysis to understand:

  • Quotes are optional in YAML. By default, "element" in JSON looks like element in YAML. There are exceptions when quotes are necessary, such as if you wish to turn a number into a string value or if you use special characters.
  • YAML is space-sensitive and depends on line breaks to indicate object structure (known as block format). Required indentation is 1 or more spaces (tabs are not allowed in YAML).

    In this example, the value of guitar is an object containing 6 key/value pairs: style, string, bridge, body, cymbal, and tunings.
  • You may also use inline format to save vertical space. Inline format looks almost identical to JSON, minus the required quotes. For example: the style object is represented as
    style: [acoustic, electric]
  • Dashes represent list items, also known as arrays. Just as in JSON, objects and arrays are frequently nested inside each other.

More YAML Tricks

Comments

YAML comments begin with # tags.

        
# Insert comment here

Anchors

YAML anchors are similar to variables in programming. When you set a value to a key, you may anchor it as well. That allows you to use that value later on in the file. Take a look at the following example, in which id has a complicated value.

        
  user:
    name: Prince Harry
    id: &id_harry 5466784563474

The value is long and difficult to remember. When you prepend the value of id with &id_harry, You set that value to the much more easily remembered id_harry. You may then call upon it later in the document as such:

        
  Harry:
    id: *id_harry

The asterisk marks the reference to the anchor that was set above.

References

The final important YAML element to learn for OpenAPI is $ref. $ref is used to reference files outside the current YAML file.

        
$ref: "#/assets/images/Harry.jpg"

The filepath of the file can be local, remote, or a URL.

Note: There is a limitation to where you are allowed to insert $ref in your OpenAPI project. More on this later.
