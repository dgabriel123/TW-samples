Introduction to JSON

JSON, short for JavaScript Object Notation, is a commonly used format for storing and moving information. It is both human-readable and easily parsed by computers, making it a natural fit for the world of APIs.

If you already have a working knowledge of JavaScript, JSON will be familiar territory, and you may skip this section. If you are new to JavaScript, consider the following example:

All JSON objects are nested inside a set of curly brackets. From this example, we can surmise the following:

  • This example contains one object, titled songs
  • songs contains 3 key-value pairs, which include the song name and the original recording artist

Key-value pairs are the drivers for JSON’s data architecture. In fact, songs is the first key-value pair in this example. In JSON, the value is allowed to be 6 types of values: a string, a digit, a boolean, an array, a nested object, or null.

String "This is a string", "This is also a string"
Digit 2, 68, 456
Boolean True, False
Array ["String string", 44, False]
Nested Object {"Nested object": "You can nest as many objects as you like"}
Null Null

YAML stands for YAML Ain’t Markup Language. It is even easier to read than JSON, and is the preferred language when working within the OpenAPI spec (particularly when using Swagger).

JSON and YAML are used interchangeably, and can be easily converted back and forth with readily available online tools, such as JSON to YAML Converter byBeautify. In this section, I will provide concurrent samples in both languages.

The syntax is not too dissimilar to JSON, and requires only a brief analysis to understand:

  • In YAML quotes are optional. By default, "element" in JSON looks like element in YAML. There are exceptions when quotes are necessary, such as if you wish to turn a digit into a string value or if you use special characters.
  • YAML is space-sensitive and depends on line breaks to indicate object structure (known as block format). Required indentation is 1 or more spaces (tabs are not allowed in YAML). In this example, the value of guitar is an object containing 6 key/value pairs: style, string, bridge, body, cymbal, and tunings.
  • You may also use inline format to save space. Inline format looks almost identical to JSON, minus the required quotes. For example: Line 3-5 can be represented as
    style: [acoustic, electric]
  • Dashes represent list items, also known as arrays. Objects and arrays are frequently nested in each other.
